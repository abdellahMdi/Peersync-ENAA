L'ENAA est un bootcamp de développement web intensif où le rythme est extrêmement soutenu. Pour éviter le décrochage, l'école s'appuie sur une forte culture d'entraide. Actuellement, la mise en relation entre les apprenants qui bloquent sur un concept (ex: la POO) et ceux qui maîtrisent le sujet (les tuteurs) se fait de manière informelle sur Discord.

La problématique : Beaucoup de demandes se perdent dans le flux de messages, et l'administration de l'ENAA ne peut pas mesurer l'impact du tutorat ni valoriser l'engagement des tuteurs volontaires.

La solution : Développer PeerSync. Pour garantir la fiabilité du système, chaque interaction (demande, session, statut) doit être un Objet PHP typé et encapsulé, persisté de manière robuste dans une base de données MySQL.
